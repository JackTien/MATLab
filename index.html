CLOUD-THREAT INTELLIGENT
APPLIANCE (CIA)

 Design Goals

In recent years, cyber security threats have become increasingly dangerous. Therefore, we develop a Cloud-threat Inspection Appliance (CIA) system to defend against emerging APT attacks. In concrete, with the advantages of hardwareassisted virtualization technology, We developed a transparent sandbox system in hypervisor (Hypervisor monitor). The security monitor module is concealed in a virtual kernel layer, and cannot be bypassed using evasion techniques. All suspicious behaviors were logged and analyzed by CIA.

 Features

• Hypervisor Sandbox: CIA uses hardware-assisted virtualization technology to build up a security monitor module in hypervisor (VMM). It concealed in virtual kernel layer and can’t be found by any evasion techniques. All the suspicious behavior can be logged and analyzed.
• Virtual Memory Forensics: Use VM introspection method to monitor malware running memory data outside the VM and analyze its system behaviors, such as process, file, registry, and network activities.

 Design Goal

IoT devices have become ubiqui tous, and they are used in our da i ly lives. These devices containing with internet communication, along with other smart functionalities, is handled by software (termed "firmware") embedded in them.
Because of there are several security breaches happening due to IoT firmware vulnerabilities, searching vulnerabilities in massive IoT ecosystems and firmware files is more crucial than ever. To address this problem, we aim to implement an
automatic IoT vulnerability probing technology by using binary analysis and symbolic execution robots methods for firmware files.
